# Brainstorm: 意义编织 (Meaning Weaver)

**Date:** 2026-03-26
**Status:** Brainstorm Complete
**Participants:** 社会学家、心理学家、市场分析师、人文学者、用户

---

## What We're Building

一个帮助用户从日常生活碎片中发现隐藏模式和自我洞察的工具。

用户通过多模态方式（文字、语音、照片）随手记录生活碎片——不是写日记，而是捕捉瞬间的想法、感受、事件。AI 定期分析这些碎片，生成"生活线索报告"，帮用户看见自己的关注点、情绪模式、价值倾向和成长轨迹。

**一句话定位：** 不是日记 App，是一面帮你看见自己的镜子。

---

## Why This Approach

### 问题背景（四角色共识）

| 视角 | 洞察 |
|------|------|
| 社会学 | 碎片化生活导致身份认同焦虑——"我到底是谁"成为普遍困惑 |
| 心理学 | 意义感缺失是注意力碎片化、决策疲劳、比较焦虑的"汇流终点" |
| 市场 | 现有日记 App 只做记录，无人做"从碎片中发现意义"，大公司不会做 |
| 人文 | 碎片化打断了人的"自我叙事"，技术应帮人重建而非进一步打碎 |

### 心理学原理支撑

- **叙事身份理论（McAdams）**：人通过"讲述自己的故事"建立连贯自我
- **元认知理论（Flavell）**：自我觉察是心理健康的核心能力
- **意义建构模型**：意义不是被发现的，而是从日常经验中被编织出来的

### 为什么现有方案不够

- **Day One / 日记类**：只存储，不洞察。需要用户有写作习惯和意愿
- **Calm / Headspace**：静态冥想，脱离真实生活场景
- **情绪追踪类**：只追踪情绪标签，不理解语境和叙事
- **ChatGPT 对话**：需要用户主动提问，不会持续跟踪和编织

## Key Decisions

### 1. 核心体验："碎片输入 → 洞察输出"

- **输入**：多模态混合——文字、语音、照片，什么方便用什么
- **输出**：周报形式——每周一份"你的生活线索"报告
- **核心价值**："看见自己"——自我洞察，而非决策建议或情绪管理

### 2. 隐私策略：本地优先

- 所有个人数据存储在用户设备本地
- AI 分析在本地进行（小模型），或用户主动选择发送到云端（加密）
- 服务器不保存任何用户内容

### 3. 产品形态：移动 App

- 手机 App，随时随地记录
- 输入极低摩擦——打开即可记录，无需分类、打标签

### 4. 设计哲学

- **反效率**：不催促用户多记录，不搞打卡机制
- **不评判**：AI 不说"你应该怎样"，只说"我注意到你在关注这些"
- **YAGNI**：MVP 不做社交、不做导出、不做模板

---

## MVP Scope

### 做的
1. 多模态碎片输入（文字 + 语音转文字 + 照片）
2. 本地数据存储
3. AI 周报生成——从一周的碎片中提炼主题、模式、洞察
4. 历史周报浏览

### 不做的
- 社交/分享功能
- 情绪标签或分类系统
- 数据导出
- 桌面端
- 打卡/streak 机制

---

## Target Users

**主要人群：** 25-40 岁城市知识工作者

- 感觉生活被日常琐事淹没，缺乏方向感
- 有反思的愿望，但没有写日记的习惯
- 对自我成长感兴趣，但厌倦了鸡汤式的自助工具
- 重视隐私，不愿意把内心想法交给大平台

---

## Resolved Questions

1. **AI 方案**：先用云端 API（如 Claude/GPT）跑通 MVP，验证洞察质量，后续再迁移到本地模型。隐私保护通过端到端加密 + 用户可控的数据传输实现。
2. **冷启动**：支持导入已有数据（微信聊天记录、备忘录、日记等），让用户第一天就能看到有意义的洞察。
3. **技术栈**：待规划阶段决定（Flutter / React Native / 原生）。

## Open Questions

1. **数据导入格式**：支持哪些数据源的导入？微信导出的技术可行性如何？
2. **周报的具体结构**：AI 生成的"生活线索报告"应该包含哪些板块？
